## 2) 返信後の深掘り質問テンプレ（要件ヒアリング）

ご返信ありがとうございます。  
認識合わせのため、以下をご教示いただけると精度高くご提案できます。

### 1. 目的・KPI
1. この案件で最も解決したい課題は何でしょうか？  
2. 成功の判断基準（例：作業時間◯%削減、ミス率低下、売上向上など）はありますか？

### 2. 現状業務・システム
3. 現在の業務フロー（手作業箇所含む）を簡単に共有いただけますか？  
4. 既存システム/利用中ツール（例：スプレッドシート、kintone、Slack、基幹システム等）は何ですか？  
5. 既存コードやドキュメントはありますか？

### 3. 技術要件
6. 希望技術スタックはありますか？（Python / Laravel / JS / AWS など）  
7. インフラ要件（AWS利用有無、リージョン、セキュリティ要件）はありますか？  
8. 外部API連携・ログイン認証・権限管理は必要ですか？

### 4. 運用・体制
9. リリース後の運用は誰が担当しますか？（内製 / 外注）  
10. 保守対応（障害時連絡、改修頻度）の希望はありますか？

### 5. スケジュール・予算
11. 希望リリース時期はいつですか？  
12. 予算レンジ（目安）があれば共有いただけますか？

必要であれば、上記をもとに  
- 最小構成（MVP）  
- 本番運用を見据えた構成  
の2パターンで提案書を作成します。

## 3) 価格提示テンプレ（30万 / 50万 / 80万）

ご要望に合わせて、3つのプランをご用意しました。  
（すべて **Python / Laravel / JS / AWS** を前提に対応可能です）

### A. ライトプラン：**30万円（税別）**
**対象**：まずは小さく始めたい / 早く動くものが欲しい  
**内容（例）**
- 要件整理（1〜2回）
- MVP開発（主要機能を最小構成で実装）
- 簡易テスト
- 初期デプロイ支援（1環境）
- 納品後サポート：2週間（軽微修正）

### B. スタンダードプラン：**50万円（税別）**
**対象**：実運用を想定した安定構成が欲しい  
**内容（例）**
- 詳細要件定義
- 本実装（主要機能＋運用に必要な周辺機能）
- エラーハンドリング / ログ整備
- AWS本番構成へのデプロイ支援
- ドキュメント整備（運用手順書）
- 納品後サポート：1か月

### C. アドバンスドプラン：**80万円（税別）**
**対象**：拡張性・セキュリティ・運用品質まで重視  
**内容（例）**
- 要件定義〜設計〜開発〜運用設計を一貫対応
- 複数システム/API連携
- 権限管理・監査ログなどの実装
- CI/CD・監視・アラート整備（必要に応じて）
- パフォーマンス改善 / セキュリティ観点レビュー
- 納品後サポート：2か月
