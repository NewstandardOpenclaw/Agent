# Client専用 AIコンサル運用キット

このディレクトリは、Discord運用で「複数エージェントが役割分担して継続改善する」ための設計書です。

## チーム構成

1. **企画 (Planner)**
   - 週次テーマ設計
   - 発信カレンダー作成

2. **執筆 (Writer)**
   - 投稿文作成（短文/長文/CTA）
   - スレッド原稿化

3. **エンジニア (Engineer)**
   - ポートフォリオ/LP改善
   - 自動化スクリプト・計測実装

4. **アナリスト (Analyst)**
   - 投稿反応の分析
   - 仮説と改善示唆

5. **改善PM (Optimizer)**
   - 全体優先順位付け
   - 次スプリントの実行計画

6. **秘書 (Secretary)**
   - リマインド
   - 進行管理・抜け漏れ防止

## Discordチャンネル運用（推奨）

- `#戦略`：方針・優先順位
- `#実行`：日々の依頼
- `#検証`：結果レビュー
- `#投資`：投資関連（作成済み）
- `#運用`：自動化ログ/定期レポート

## 自動化（cron）案

- 毎日 09:00: 今日の投稿案3本
- 毎日 20:00: 日次レビュー + 明日の改善1つ
- 毎週 月曜 09:00: 週間戦略アップデート

## 導入手順

1. この設計をベースにOpenClawで各タスクを定期実行
2. 必要に応じてサブエージェントを並列起動
3. `operations/runbook.md` のテンプレで毎日運用
