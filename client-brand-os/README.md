# client-brand-os

clientを「エンジニアとして売る」ための実行プロジェクト。

## 目的
3週間で「相談が来る状態」を作る。

## North Star KPI
- 週あたりの指名相談数（DM/Discord）

## 3週間スプリント

### Week 1: 証拠作り
- 実績が伝わるGitHub成果物を1つ公開
- READMEに課題→施策→結果を明記

### Week 2: 発信運用
- 週5本（実績/学び/問題提起）を投稿
- すべてにCTAを付与

### Week 3: 導線最適化
- プロフィール・固定投稿・DM導線を改善
- 相談導線を1つに統一（フォームorDM）

## 役割分担（サブエージェント想定）
- Planner: テーマ決定
- Writer: 投稿作成
- Engineer: 成果物実装
- Analyst: 反応分析
- Secretary: 進行管理

## Daily Ops
- 朝: 投稿案3本
- 夜: KPIレビュー + 改善1つ
