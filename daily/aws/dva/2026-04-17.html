Topic 1: SQS 可視性タイムアウトと再試行制御重複処理の抑制

定義

Visibility Timeoutは、SQSコンシューマーがメッセージを受信した後に他コンシューマーから一時的に見えなくする時間。処理完了前に期限切れすると同じメッセージが再配信される。

重要性

  • 短すぎると処理中メッセージが再配信され、重複実行や副作用を起こしやすい。
  • 長すぎると障害時のリカバリが遅れ、キュー滞留が悪化する。

使い分け

  • 処理時間が読める: 平均処理時間 + バッファでVisibility Timeoutを固定設定。
  • 処理時間が揺れる: LambdaやWorkerで ChangeMessageVisibility を使って延長。
  • 恒久失敗を隔離したい: maxReceiveCount + DLQで毒メッセージを切り離す。

誤解

  • 「SQSなら自動でExactly Once」→ 標準キューはAt-Least-Once。冪等化が必須。
  • 「Visibility Timeoutを長くすれば安全」→ 障害時の再処理遅延を招く。

3分要点

  • Visibility Timeoutは「重複」と「復旧速度」のバランス調整ノブ。
  • DVAではDLQと冪等設計をセットで答えると強い。
  • 処理時間が変動するなら動的延長を検討する。

参照: https://docs.aws.amazon.com/AWSSimpleQueueService/latest/SQSDeveloperGuide/sqs-visibility-timeout.html

Topic 2: X-Ray トレースでLambdaボトルネックを特定可観測性強化

定義

AWS X-Rayは、分散システムのリクエストをトレースし、各サービス区間の遅延・エラーを可視化する。Lambda、API Gateway、DynamoDBなどと連携可能。

重要性

  • 「遅い」の原因をコード・外部呼び出し・コールドスタートまで分解できる。
  • DVAの監視系問題で、CloudWatch Logs単独との差分が問われる。

使い分け

  • 個別関数の例外調査: CloudWatch Logsで詳細ログ確認。
  • 経路全体の遅延調査: X-Ray Service Mapでボトルネック箇所を把握。
  • 高トラフィック環境: サンプリング率を調整してコストと粒度を最適化。

誤解

  • 「X-Rayがあればログ不要」→ 例外の文脈や入力値確認はログが必要。
  • 「全件トレースが常に最適」→ コスト増とノイズ増を招く。

3分要点

  • X-Rayは“どこで遅いか”、Logsは“なぜ失敗したか”に強い。
  • Lambda/API Gateway連携時はトレース有効化漏れを先に疑う。
  • DVAでは可観測性ツールの役割分担を意識して選択する。

参照: https://docs.aws.amazon.com/xray/latest/devguide/xray-services-lambda.html

Topic 3: Lambda Function URL と API Gateway の選択基準公開API設計

定義

Lambda Function URLはLambdaへ直接HTTPSエンドポイントを付与する機能。API Gatewayは認証・スロットリング・変換・ステージ管理を含むフル機能APIフロント。

重要性

  • 要件に対して過剰構成/不足構成を避け、コストと運用負荷を最適化できる。
  • DVAでは「最小構成で素早く公開」か「制御機能重視」かの見極めが頻出。

使い分け

  • 内部ツール・単機能Webhook: Function URL（シンプル・低オーバーヘッド）。
  • 複数エンドポイント・利用者別制御: API Gateway（認証/制限/モニタリングが豊富）。
  • 将来拡張見込みあり: 初期からAPI Gatewayで設計して移行コストを抑える。

誤解

  • 「Function URLは常にAPI Gatewayの代替」→ 高度な認可やUsage Planは代替できない。
  • 「API Gatewayは重いから使うべきでない」→ 要件があるなら標準解で、運用の安定性が上がる。

3分要点

  • Function URLは“最短公開”、API Gatewayは“制御付き公開”。
  • 認証・流量制御・複数ルート要件があればAPI Gatewayが本命。
  • DVAは「要件に対して十分な機能」を選ぶ問題だと捉える。

参照: https://docs.aws.amazon.com/lambda/latest/dg/urls-configuration.html
