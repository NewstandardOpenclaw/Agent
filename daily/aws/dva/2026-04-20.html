Topic 1: STS AssumeRole + セッションポリシーで一時的に権限を絞る最小権限を実行時に強制

定義

AssumeRole は STS で一時クレデンシャルを発行して別ロールを引き受ける仕組み。セッションポリシー を付与すると、そのセッション中だけ追加で許可範囲を狭められる。

重要性

  • 同じ実行ロールでも、処理単位でアクセス範囲を下げられ、誤操作・侵害時の影響を最小化できる。
  • DVAで頻出の「クロスアカウントアクセス」「一時認可」の設計問題で説明しやすい。

使い分け

  • 恒常的な権限: IAMロールのベースポリシーで管理。
  • リクエストごとに制限: AssumeRole時にセッションポリシーを渡して範囲縮小。
  • 利用者属性で制御: タグベースABAC + セッショントークンのタグ連携。

誤解

  • 「セッションポリシーは権限を増やせる」→ 増やせない。ベース権限を“さらに絞る”だけ。
  • 「長期アクセスキーの代替は不要」→ 一時クレデンシャル化が原則で、長期キーは極小化すべき。

3分要点

  • AssumeRoleは“誰として実行するか”を切り替える機能。
  • セッションポリシーは“その回だけ何をさせるか”を狭める機能。
  • DVA回答では「最小権限」「短命クレデンシャル」をセットで述べる。

参照: https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_temp_control-access_assumerole.html

Topic 2: Lambda × SQS の Partial Batch Response再試行を失敗メッセージに限定

定義

Partial Batch Response は、SQSトリガーのLambdaでバッチ中の失敗メッセージIDだけ返し、そのメッセージのみ再処理対象にする機能。

重要性

  • 1件失敗で全件再実行される無駄を避け、重複実行コストとレイテンシを削減できる。
  • 高スループット処理での“毒メッセージ”混入時に、健全メッセージの進行を止めにくい。

使い分け

  • 独立処理できるメッセージ: Partial Batch Response を有効化。
  • 順序保証が必要（FIFO）: MessageGroup単位の設計と可視性タイムアウトを慎重に調整。
  • 恒久失敗想定: DLQ へ退避し、再処理フローを別系統で運用。

誤解

  • 「有効化すれば冪等性は不要」→ 重複配信は残るため、冪等設計は依然必須。
  • 「DLQがあれば監視不要」→ ApproximateAgeOfOldestMessage などの監視は必要。

3分要点

  • 失敗したレコードだけ再試行させるのがPartial Batch Responseの本質。
  • 可視性タイムアウト・バッチサイズ・DLQをセットで設計する。
  • DVAでは「再試行範囲の最適化」を明確に言えると強い。

参照: https://docs.aws.amazon.com/lambda/latest/dg/with-sqs.html#events-sqs-batchfailurereporting

Topic 3: Lambda のエイリアス + CodeDeploy Canary デプロイ安全に段階リリース

定義

Lambdaエイリアス は特定バージョンへの参照名。CodeDeploy Canary は新バージョンへトラフィックを段階移行し、CloudWatch Alarm 異常時に自動ロールバックする方式。

重要性

  • 全量切替の障害リスクを下げ、本番での回復時間（MTTR）を短縮できる。
  • DVAで問われる「安全なリリース戦略」「自動ロールバック」の定番解。

使い分け

  • 低リスク変更: All-at-once で迅速展開。
  • 中〜高リスク変更: Canary10Percent5Minutes など段階移行。
  • 業務影響が大きいAPI: PreTraffic/PostTraffic フックで追加検証。

誤解

  • 「バージョン管理だけで十分」→ エイリアス運用とアラーム連携がないと自動退避できない。
  • 「Canaryは遅いだけ」→ 障害の早期検知で全体影響を大幅に削減できる。

3分要点

  • Lambdaは“バージョン固定 + エイリアス経由”が本番運用の基本。
  • CodeDeployで段階配信し、Alarmで失敗時ロールバックする。
  • DVA回答では「可用性優先のデプロイ設計」を具体化して語る。

参照: https://docs.aws.amazon.com/lambda/latest/dg/configuring-alias-routing.html
