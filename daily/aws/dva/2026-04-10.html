Topic 1: IAMロールの信頼ポリシーと権限ポリシーの役割分離認可の土台

定義

信頼ポリシー（Trust Policy）は「誰がそのロールを引き受けられるか」を定義し、権限ポリシー（Permissions Policy）は「引き受けた後に何ができるか」を定義する。DVAではこの2層を混同しないことが重要。

重要性

  • Lambda・ECS・CodeBuild などサービス連携時の403は、信頼/権限の切り分け不足で発生しやすい。
  • 試験では「最小権限かつ安全なサービス間アクセス」の設問が頻出。

使い分け

  • サービスにロールを渡す: 信頼ポリシーで principal をサービス名に限定（例: lambda.amazonaws.com）。
  • 実行時にS3へアクセス: 権限ポリシーで対象バケットとアクションを最小化。
  • クロスアカウント: 信頼ポリシーに相手アカウントを明示、必要なら ExternalId を使用。

誤解

  • 「Actionを許可したのに動かない」→ 信頼ポリシーに assume role 元が含まれていない可能性。
  • 「信頼ポリシーにS3権限を書けばよい」→ 誤り。操作権限は権限ポリシー側。

3分要点

  • 信頼ポリシー=入口、権限ポリシー=入った後の行動範囲。
  • まず「誰が使うか」を固定し、次に「何をするか」を最小化。
  • 設問で悩んだら“AssumeRoleできるか”を最初に確認。

参照: https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_terms-and-concepts.html

Topic 2: SQS可視性タイムアウトとLambdaバッチ失敗時の再処理重複処理を防ぐ

定義

SQS の可視性タイムアウトは、受信後のメッセージを一時的に他コンシューマから見えなくする時間。Lambda 連携では処理時間と再試行戦略に合わせて設定し、重複処理や早すぎる再配信を防ぐ。

重要性

  • 短すぎると処理中に再配信され、二重書き込みが発生する。
  • 長すぎると失敗時の再処理が遅れ、復旧時間が伸びる。

使い分け

  • 短時間処理: Lambda timeoutの数倍を目安に可視性タイムアウト設定。
  • 部分失敗があり得る: partial batch response を有効化し、失敗メッセージのみ再処理。
  • 恒久失敗対策: maxReceiveCount + DLQ で無限再試行を防止。

誤解

  • 「可視性タイムアウト=削除までの猶予」→ 誤り。削除は明示的 DeleteMessage が必要。
  • 「Lambdaなら重複実行は起きない」→ 誤り。少なくとも1回配信前提で冪等設計が必要。

3分要点

  • SQS + Lambda は at-least-once 前提で設計する。
  • 可視性タイムアウトは“長すぎず短すぎず”が鍵。
  • DVAでは partial batch response を選べると得点しやすい。

参照: https://docs.aws.amazon.com/lambda/latest/dg/with-sqs.html

Topic 3: API Gateway Usage Plan と API Key の実務的な位置づけ制御と保護

定義

API Gateway の Usage Plan はクライアント単位のレート制御（throttle）と利用量制限（quota）を管理する仕組み。API Key は“識別子”であり、認証・認可の代替ではない。

重要性

  • 特定クライアントの過剰呼び出しで全体性能が劣化する事態を防げる。
  • DVAでは「認証はCognito/IAM、利用制御はUsage Plan」という責務分離が問われる。

使い分け

  • SaaSのプラン別制御: Free/Proで異なる quota と burst limit を設定。
  • 内部向けAPI: IAM認証 + Usage Planで誤爆トラフィックを抑制。
  • 外部公開API: WAFと併用し、攻撃耐性と公平利用を両立。

誤解

  • 「API Keyがあれば安全」→ 誤り。漏洩すると誰でも使えるため認証の代替にならない。
  • 「Usage Planで厳密課金できる」→ 誤り。運用上の制御用途で、課金の最終根拠には不向き。

3分要点

  • API Keyは“誰かを識別する札”、権限管理は別レイヤー。
  • Usage Planは安定運用と公平利用のためのガードレール。
  • 試験では「セキュリティ要件」と「スロットリング要件」を分離して判断。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/api-gateway-api-usage-plans.html
