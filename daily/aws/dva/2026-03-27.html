Topic 1: DynamoDB の整合性モデルとアクセス設計 頻出

定義

DynamoDB はデフォルトで 結果整合性 (Eventually Consistent Read)。強い整合性 (Strongly Consistent Read) はテーブル/LSI のみで利用でき、GSI やグローバルテーブルには適用されない。

重要性

  • 「書き込み直後の読み取りで値が見えない」系の不具合を、仕様かバグか切り分けできる。
  • DVA では「整合性要件」と「コスト/レイテンシ」のトレードオフ問題が頻出。

使い分け

  • 強い整合性: 決済直後の残高確認など、最新値が必須な場面。
  • 結果整合性: タイムライン表示、集計、レコメンド等の読み取り中心処理。
  • 補助策: 書き込み後の read-back は primary key で取得、必要なら短いリトライを設計。

誤解

  • 「DynamoDB は常に即時反映」→ 誤り。整合性モデルを意識しないと再現しづらい障害になる。
  • 「GSI でも Strongly Consistent Read 可能」→ 誤り。GSI は結果整合性のみ。

3分要点

  • 強い整合性は“必要最小限”で使う（コストと遅延が増える）。
  • GSI/グローバルテーブルに最新性を期待しすぎない。
  • 設計時に「どの画面が最新必須か」を先に決める。

参照: https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/HowItWorks.ReadConsistency.html

Topic 2: Lambda の同時実行制御（Reserved / Provisioned Concurrency）

定義

Reserved Concurrency は関数に割り当てる同時実行数の上限兼確保枠。Provisioned Concurrency はコールドスタートを抑えるために事前初期化された実行環境。

重要性

  • スパイクアクセス時のレイテンシ悪化と、下流DB過負荷の両方を制御できる。
  • DVA で「安定応答時間を要求される API」の設計問題に直結。

使い分け

  • Reserved: 他関数に同時実行を奪われないよう守る / 暴走時の上限設定。
  • Provisioned: レイテンシ要件が厳しい同期API（例: 認証、決済前段）。
  • 両方併用で「上限管理 + 低遅延」を同時に達成。

誤解

  • 「Provisioned を設定すればスロットリングしない」→ 誤り。リクエスト急増で上限を超えれば別途制約に当たる。
  • 「Reserved は高速化機能」→ 誤り。主目的は容量隔離と上限管理。

3分要点

  • 低遅延 = Provisioned、隔離/制御 = Reserved と覚える。
  • 要件を「p95 レイテンシ」「最大RPS」で分解して設定する。
  • CloudWatch で Throttles / ConcurrentExecutions を必ず観測。

参照: https://docs.aws.amazon.com/lambda/latest/dg/configuration-concurrency.html

Topic 3: API Gateway + Lambda Authorizer + Cognito の責務分離

定義

API Gateway では、認証/認可を Cognito User Pool Authorizer または Lambda Authorizer で実施できる。Cognito は標準JWT運用向き、Lambda Authorizer は独自ルール実装向き。

重要性

  • 「認証基盤を自作すべきか」の判断を誤ると保守コストが急増。
  • DVA の設問で、最小実装で安全性を満たす選択肢を見抜く鍵になる。

使い分け

  • Cognito Authorizer: モバイル/Web の一般的ログイン、OIDC/JWT 前提。
  • Lambda Authorizer: 社内独自トークン、外部権限DB照会、条件付きポリシー発行。
  • キャッシュ有効時は認可判定の更新遅延を考慮（TTL設計）。

誤解

  • 「Lambda Authorizer の方が常に柔軟で正解」→ 誤り。柔軟性と引き換えに運用負荷が増える。
  • 「認証と認可は同じ」→ 誤り。認証は本人確認、認可は操作許可。

3分要点

  • まず Cognito で成立するか検討、足りない要件だけ Lambda で補う。
  • Authorizer キャッシュTTLと権限変更反映の整合を設計する。
  • 最小権限（least privilege）を IAM と API の両面で適用。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-use-lambda-authorizer.html / https://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-integrate-with-cognito.html
