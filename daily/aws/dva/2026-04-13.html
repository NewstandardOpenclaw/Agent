Topic 1: STS AssumeRole と External ID の実務設計クロスアカウントの基本

定義

STS AssumeRole は、別アカウントのIAMロールを一時的に引き受けて短期クレデンシャルを取得する仕組み。External ID は第三者SaaS連携でConfused Deputyを防ぐ追加条件として使う。

重要性

  • 長期アクセスキーを配布せずに連携でき、漏えい面積を最小化できる。
  • DVAでは「安全なクロスアカウント連携」の選択肢として頻出。

使い分け

  • 自社アカウント間連携: 信頼ポリシー + 最小権限でAssumeRole。
  • 外部ベンダー連携: External ID を trust policy 条件に必ず追加。
  • 短時間作業: セッション時間を短くし、CloudTrailで追跡可能性を確保。

誤解

  • 「External ID は暗号キー代わり」→ 秘密情報というより“誤用防止の識別子”。
  • 「AssumeRoleなら何でも許可される」→ 実効権限はロールポリシーとセッションポリシーで制限される。

3分要点

  • 原則は「長期鍵を渡さず、短期権限を渡す」。
  • 外部連携では External ID が事故防止の要。
  • DVAの正答軸は“最小権限 + 追跡可能 + 短期化”。

参照: https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create_for-user_externalid.html

Topic 2: Cognito User Pool と Identity Pool の役割分担認証とAWS認可を分離

定義

User Pool はユーザー登録・ログイン・トークン発行を担う認証基盤。Identity Pool はそのIDトークン等を元に、IAMロール経由でAWSリソースアクセス用の一時クレデンシャルを払い出す。

重要性

  • アプリ認証とAWSアクセス制御を分離でき、設計が明確になる。
  • モバイル/SPAでS3直アップロードなどを安全に実装しやすい。

使い分け

  • ログイン機能が必要: User Pool を中心に設計。
  • AWSリソースへの直接アクセスが必要: Identity Pool を追加してロールマッピング。
  • ゲスト利用を許可: unauthenticated role を最小権限で分離。

誤解

  • 「User Pool だけでS3アクセス制御まで完結」→ IAMクレデンシャル発行はIdentity Poolの責務。
  • 「Identity Pool は認証DBを持つ」→ 認証そのものは外部IdPまたはUser Pool側。

3分要点

  • User Pool = 認証、Identity Pool = AWS認可用の橋渡し。
  • ロールマッピング条件（sub/group）は試験で狙われる。
  • “誰か”の証明と“何ができるか”を分けて考えると迷わない。

参照: https://docs.aws.amazon.com/cognito/latest/developerguide/cognito-identity.html

Topic 3: IAM Policy Evaluation（Allow/Deny 境界）本番トラブルの即断力

定義

IAMの評価は「明示的Denyが最優先」。その上で identity-based policy、resource-based policy、permission boundary、SCP などの交差で最終可否が決まる。

重要性

  • “なぜ許可されないか”を構造的に切り分けられる。
  • DVAのシナリオ問題で、正しい制約レイヤーを選ぶ鍵になる。

使い分け

  • チーム横断で上限を固定: Permission Boundary で委任範囲を制限。
  • 組織全体で禁止: SCP でアカウント横断のガードレール化。
  • サービス共有: Resource-based policy でPrincipalを明示。

誤解

  • 「Allowを増やせば通る」→ どこかにDenyがあれば必ず拒否。
  • 「SCPは権限を付与できる」→ SCPは“許可”ではなく“上限/禁止”を定義する。

3分要点

  • まずExplicit Denyの有無を確認。
  • 次に“どのレイヤーが制限しているか”を特定。
  • DVAでは「付与」と「上限制約」を混同しないことが得点差になる。

参照: https://docs.aws.amazon.com/IAM/latest/UserGuide/reference_policies_evaluation-logic.html
