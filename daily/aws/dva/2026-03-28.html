Topic 1: SQS FIFO の重複排除と順序保証 本番設計直結

定義

FIFO キューは MessageGroupId 単位で順序を保証し、重複排除ID（Deduplication ID） により一定時間（5分）内の重複メッセージを抑制する。

重要性

  • 注文処理・在庫更新など「同じイベントを二重反映できない」処理で必須。
  • DVAでは「標準キューかFIFOか」「重複をどこで防ぐか」の設問がよく出る。

使い分け

  • FIFO: 順序/重複抑制が必要（例: 会計、台帳更新）。
  • Standard: 高スループット優先、順序不要（例: ログ集約、非同期通知）。
  • 順序はキュー全体ではなく MessageGroupId ごと に守られる点を意識する。

誤解

  • 「FIFOなら絶対に重複ゼロ」→ 誤り。アプリ側の冪等設計は引き続き必要。
  • 「FIFOは全メッセージ1本の順序」→ 誤り。グループ単位で並列処理できる。

3分要点

  • 順序保証の単位は MessageGroupId。
  • 重複排除は万能ではないので、DB更新は冪等に設計。
  • 要件が順序不要なら Standard を選び、コスト/性能を優先。

参照: https://docs.aws.amazon.com/AWSSimpleQueueService/latest/SQSDeveloperGuide/FIFO-queues.html

Topic 2: API Gateway のスロットリングと Usage Plan

定義

API Gateway は レート制限（rate） と バースト制限（burst） で流量を制御でき、API キーを使う場合は Usage Plan でクライアントごとの利用上限を管理できる。

重要性

  • 下流の Lambda / DB 保護に直結し、障害時の連鎖を抑えられる。
  • DVAでは「最小変更で過負荷対策する方法」を問う設問で頻出。

使い分け

  • ステージ/メソッドのスロットリング: 全体防御の基本設定。
  • Usage Plan + API Key: 顧客ごとの契約帯域や試用枠管理。
  • AWS WAF: 悪意トラフィックやL7攻撃の遮断を追加。

誤解

  • 「APIキーは認証そのもの」→ 誤り。認証は IAM/Cognito/Lambda Authorizer などで別途実施。
  • 「スロットリングで100%防げる」→ 誤り。再試行戦略・バックオフも設計に必要。

3分要点

  • まず API Gateway で入口制御し、下流を守る。
  • 契約差分は Usage Plan で分離。
  • 認証・認可と流量制御は目的が違うので混同しない。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/api-gateway-request-throttling.html

Topic 3: CodeDeploy Blue/Green（ECS/Lambda）のロールバック戦略

定義

Blue/Green デプロイは新旧環境を並行させ、トラフィックを段階移行する方式。CodeDeploy と CloudWatch Alarm を組み合わせると、異常検知時に自動ロールバックできる。

重要性

  • 本番リリース時のリスクを下げ、復旧時間（MTTR）を短縮できる。
  • DVAでよく問われる「ダウンタイム最小」「失敗時の即時復旧」を満たす代表解。

使い分け

  • Canary/Linear: 影響を小さく観測しながら移行。
  • All-at-once: 迅速だが失敗時インパクト大。短時間停止許容時のみ。
  • ロールバック判定は 5xx、レイテンシ、ビジネスメトリクスの複合で設計する。

誤解

  • 「Blue/Greenなら自動で安全」→ 誤り。アラーム条件が甘いと不良が流れる。
  • 「ロールバックは手動で十分」→ 誤り。夜間障害では自動化が被害を最小化する。

3分要点

  • Blue/Green + Alarm + Auto rollback をセットで覚える。
  • トラフィック移行戦略はサービス特性で選ぶ。
  • 成功判定は技術指標だけでなく業務指標も見る。

参照: https://docs.aws.amazon.com/codedeploy/latest/userguide/deployment-steps-lambda.html / https://docs.aws.amazon.com/codedeploy/latest/userguide/deployments-create-ecs.html
