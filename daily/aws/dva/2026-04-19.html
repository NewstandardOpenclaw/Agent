Topic 1: API Gateway Lambda オーソライザー（JWT/Cognito）の設計認証をアプリ外で標準化

定義

Lambda オーソライザー は API Gateway でリクエストごとにトークンや署名を検証し、許可/拒否を判定する仕組み。HTTP API では JWT オーソライザー（Cognito/外部IdP）も使える。

重要性

  • 各Lambda関数に認証ロジックを重複実装せず、保守負荷を下げられる。
  • 認証失敗をAPI入口で止めるため、バックエンドコストと攻撃面を減らせる。

使い分け

  • 標準的なJWT検証: HTTP API の JWT オーソライザーで低レイテンシ構成。
  • カスタム判定が必要: Lambda オーソライザーでDB参照や追加条件を実装。
  • 高トラフィック: キャッシュ設定を調整し、認証遅延を抑える。

誤解

  • 「認証はバックエンド側だけで十分」→ 入口側で拒否しないと不要実行が増える。
  • 「キャッシュは常に有効化すべき」→ ユーザー権限の即時反映が必要な場合はTTLを短くする。

3分要点

  • 認証はAPI Gateway側で共通化するとDVAの設計問題で強い。
  • JWT標準ならマネージド機能、独自要件ならLambdaオーソライザー。
  • 性能と即時性のトレードオフは「キャッシュTTL」で説明する。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/http-api-jwt-authorizer.html

Topic 2: DynamoDB Conditional Write で楽観ロック同時更新でデータ破壊を防止

定義

Conditional Write は DynamoDB の PutItem/UpdateItem に条件式を付け、条件成立時のみ更新を許可する機能。バージョン属性を使うと楽観ロックを実現できる。

重要性

  • 複数クライアントの同時更新で「最後の書き込み勝ち」による欠落を防げる。
  • 在庫・ポイント・状態遷移など整合性重視ユースケースで必須。

使い分け

  • 単一アイテム更新: ConditionExpression で軽量に制御。
  • 複数アイテム整合: TransactWriteItems を使い原子的に更新。
  • 競合が多い: リトライ戦略＋指数バックオフを組み合わせる。

誤解

  • 「DynamoDBは常に強整合だから競合しない」→ 更新競合は発生するため条件付き更新が必要。
  • 「失敗=システム障害」→ ConditionalCheckFailedException は設計どおりの競合通知。

3分要点

  • 競合制御は ConditionExpression が基本。
  • 複数レコードの一貫性はトランザクションを使う。
  • DVAでは「競合をどう扱うか」の説明力が得点差になる。

参照: https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/Expressions.ConditionExpressions.html

Topic 3: X-Ray + CloudWatch Logs Insights で障害切り分け分散トレースで根本原因へ最短到達

定義

AWS X-Ray はリクエスト経路をトレースし、遅延やエラー箇所を可視化する。CloudWatch Logs Insights はログをクエリ分析し、異常パターンを抽出する。

重要性

  • 「遅いが原因不明」を、依存サービス単位で分解して調査できる。
  • DVA頻出のサーバーレス障害対応（API Gateway→Lambda→DynamoDB）で有効。

使い分け

  • ボトルネック位置の特定: X-Ray のサービスマップとセグメント分析。
  • 特定エラーの母集団分析: Logs Insights で期間・条件を絞って集計。
  • 運用自動化: メトリクスフィルタ + Alarm で早期検知。

誤解

  • 「X-Rayを有効化すれば全ログ不要」→ 事象の文脈や入力値確認にはログが必要。
  • 「Logs Insightsは遅い」→ クエリ対象期間とフィールド絞り込みで高速化できる。

3分要点

  • 経路はX-Ray、詳細はLogs Insightsで補完する。
  • 調査の順序は「症状確認→経路特定→ログ深掘り」が効率的。
  • DVAでは監視サービスの役割分担を答えられると強い。

参照: https://docs.aws.amazon.com/xray/latest/devguide/xray-services-lambda.html
