Topic 1: SQS + Lambda の Partial Batch Response再処理の無駄を削減

定義

Partial Batch Response は、SQSイベントでLambdaが受けた複数メッセージのうち「失敗したレコードだけ」を再試行対象として返せる機能。成功分まで巻き戻さない。

重要性

  • 1件の不正データでバッチ全体を再処理するコストを防げる。
  • 重複処理リスクを下げ、Idempotency実装の負担を減らせる。

使い分け

  • 正常/異常が混在するキュー: Partial Batch Response を有効化して失敗IDだけ返す。
  • 完全原子性が必要: あえてバッチ全失敗として扱い、順序一貫性を優先する。
  • 恒久的失敗が多い: maxReceiveCount + DLQ を組み合わせ、毒メッセージを隔離する。

誤解

  • 「Lambda が例外を投げれば自動で部分再試行される」→ 失敗メッセージIDを明示返却しないと全体再試行になる。
  • 「Partial を使えば重複はゼロ」→ 受信保証は at-least-once。重複前提設計は依然必須。

3分要点

  • 要点は「失敗だけ戻す」こと。
  • DLQ設計とセットで初めて運用が安定する。
  • DVAでは“再処理効率”と“重複耐性”の両立を問われる。

参照: https://docs.aws.amazon.com/lambda/latest/dg/with-sqs.html#events-sqs-batchfailurereporting

Topic 2: API Gateway Lambda オーソライザーのキャッシュ設計認可の遅延とコストを制御

定義

Lambda Authorizer Cache は、同一トークン/キーに対する認可結果をTTL付きで再利用する機能。毎回認可Lambdaを呼ばずに判定できる。

重要性

  • 認可Lambdaの呼び出し回数を削減し、API全体のレイテンシとコストを下げる。
  • 高頻度APIで認可がボトルネックになるのを防ぐ。

使い分け

  • トークン更新が少ないB2B API: 長めTTLで性能優先。
  • 権限変更が頻繁: 短TTLまたはキャッシュ無効化運用を準備。
  • ユーザー属性で細かく制御: キャッシュキー設計を誤ると権限漏れするため、identity source を厳密に定義。

誤解

  • 「キャッシュを長くすれば常に正解」→ 権限剥奪の反映遅延でセキュリティ事故になる。
  • 「JWT Authorizer と同じ挙動」→ 検証主体や設定項目が異なるため要件別に選ぶ。

3分要点

  • TTLは性能と即時反映のトレードオフ。
  • identity source 設計ミスは認可バグ直結。
  • DVA問題では「低遅延」と「権限変更の即時性」どちら重視かを先に読む。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-use-lambda-authorizer.html

Topic 3: CodeDeploy Canary / Linear デプロイ（Lambda）安全に本番反映

定義

Lambdaのトラフィックシフトを Canary / Linear で段階的に実行し、CloudWatch Alarm 連携で異常時自動ロールバックするデプロイ方式。

重要性

  • 一括切替で障害を全量拡散するリスクを抑えられる。
  • 「デプロイ自動化 + 失敗時復旧」を試験で問われやすい。

使い分け

  • 影響が大きい変更: Canary（例: 10%→残り）で早期検知を優先。
  • 安定的に段階展開したい: Linear（例: 10%ずつ）で観測しながら進める。
  • 短時間メンテで即切替: All-at-once を使うが、ロールバック前提の監視を強化。

誤解

  • 「バージョン発行なしで段階配信できる」→ Alias + Version 運用が前提。
  • 「CloudWatch Alarm は任意」→ 自動ロールバックの要。未設定だと“失敗を止められない”。

3分要点

  • 安全配信は「段階配信 + 監視 + 自動戻し」の3点セット。
  • Canaryは初動確認、Linearは継続観測に強い。
  • DVAでは CodeDeploy 設定だけでなく失敗時挙動まで説明できると強い。

参照: https://docs.aws.amazon.com/codedeploy/latest/userguide/deployment-configurations.html
