Topic 1: API Gateway カスタムオーソライザー（Lambda Authorizer）と Cognito Authorizer の使い分け認可設計の頻出論点

定義

Cognito Authorizer は Cognito User Pool 発行トークンを検証し、API Gateway で認可判定する仕組み。Lambda Authorizer は任意ロジック（JWT検証、外部ID基盤、独自ルール）で IAM ポリシーを返す。

重要性

  • 認証・認可の責務を API 前段で統一でき、バックエンド実装の重複を減らせる。
  • 要件に応じて「運用容易性」か「柔軟性」かを正しく選ぶことが DVA で問われる。

使い分け

  • Cognito Authorizer: ユーザー認証が Cognito 中心で、標準的な JWT 検証をシンプルに実装したい。
  • Lambda Authorizer: 複数 IdP の統合や、属性・時間帯・テナント単位の独自判定が必要。
  • 注意: Lambda Authorizer は柔軟だが遅延・運用コストが増えやすい。

誤解

  • 「Lambda Authorizer の方が常に上位互換」→ 誤り。要件が単純なら Cognito Authorizer の方が安全で軽い。
  • 「認可は Lambda 本体でやれば十分」→ 誤り。前段で落とすとセキュリティとコスト両面で有利。

3分要点

  • 標準JWTなら Cognito、独自判定なら Lambda Authorizer。
  • 柔軟性が上がるほど、遅延と運用負荷は増える。
  • DVAは「最小構成で要件を満たす選択」が得点源。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-use-lambda-authorizer.html

Topic 2: DynamoDB TTL・Streams・Lambda の非同期データ連携

定義

TTL は有効期限切れアイテムを自動削除する機能。DynamoDB Streams はテーブル変更イベントを時系列で配信し、Lambda 連携で派生処理（集計・通知・監査）を実行できる。

重要性

  • 本番でよくある「期限付きデータの掃除」と「変更契機処理」をサーバレスで実現できる。
  • 試験では TTL 削除の性質（即時削除ではない）と Streams の用途を混同させる問題が多い。

使い分け

  • TTL: セッションやワンタイムデータなど、期限到来後に自動整理したいデータ。
  • Streams + Lambda: 更新イベントをトリガーに別テーブル同期や通知をしたい場合。
  • 組み合わせ: TTL 対象データでも、削除をトリガーに後続処理が必要なら Streams 設計を追加。

誤解

  • 「TTL 時刻を過ぎたら即時で消える」→ 誤り。削除は非同期で遅延し得る。
  • 「Streams があれば履歴保存は不要」→ 誤り。保持期間外の監査要件は別保存が必要。

3分要点

  • TTL は“期限管理”、Streams は“変更イベント連携”。
  • TTL 削除はベストエフォートで即時保証ではない。
  • DVAでは削除タイミング要件と連携要件の切り分けが鍵。

参照: https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/TTL.html

Topic 3: SQS 長時間ポーリング・可視性タイムアウト・DLQ の設計セットキュー処理の安定化

定義

長時間ポーリングは空レスポンスを減らしコストを下げる受信方式。可視性タイムアウトは処理中メッセージの再配信抑制時間。DLQは最大受信回数超過メッセージの退避先。

重要性

  • 重複処理・取りこぼし・無限リトライを避け、非同期システムを安定運用できる。
  • DVA で頻出の「何を調整すれば再処理を減らせるか」を一気に解ける。

使い分け

  • 長時間ポーリングを有効化: 空ポーリングを削減し、受信効率を上げたい。
  • 可視性タイムアウトを延長: 処理時間が長く、途中で再配信される問題を抑えたい。
  • DLQ を設定: 毒メッセージを本流から隔離し、運用で再処理判断したい。

誤解

  • 「可視性タイムアウトを長くすれば安全」→ 誤り。障害時の復旧検知が遅くなる。
  • 「DLQ に入ったら自動復旧する」→ 誤り。再処理フローは別途設計が必要。

3分要点

  • 長時間ポーリング=コスト最適化、可視性=重複制御、DLQ=失敗隔離。
  • 3点セットで調整しないと片手落ちになる。
  • DVAはパラメータ変更の効果を問う問題が多い。

参照: https://docs.aws.amazon.com/AWSSimpleQueueService/latest/SQSDeveloperGuide/sqs-visibility-timeout.html
