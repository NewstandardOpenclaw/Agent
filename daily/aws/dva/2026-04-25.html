Topic 1: STS AssumeRole と AssumeRoleWithWebIdentity の選択一時認証設計

定義

AssumeRoleはAWS IAMユーザー/ロールから別ロールの一時クレデンシャルを取得する方式。AssumeRoleWithWebIdentityはOIDC/Social IdPトークンを使って、長期キーなしで一時クレデンシャルを得る方式。

重要性

  • DVAでは「長期アクセスキーを配布しない」設計判断が高頻度で出題される。
  • モバイル/SPA/CIの認証方式を誤ると、運用負荷と漏えいリスクが急増する。

使い分け

  • AWS内の権限委譲: AssumeRole（アカウント間アクセス、運用ロール切替）。
  • 外部IdP連携のフロントアプリ: AssumeRoleWithWebIdentity（OIDCトークン前提）。
  • サーバー常駐でIAMロール付与可: STS明示呼び出しより、実行環境ロール優先。

誤解

  • 「WebIdentityならIAM設定不要」→ 信頼ポリシーでiss/aud/sub条件を厳密化する必要がある。
  • 「一時認証だから最小権限は不要」→ 有効期限が短くても過剰権限は事故要因。

3分要点

  • 長期キー配布を避けるのが第一原則。
  • AWS内委譲か外部IdP連携かでAPIを選ぶ。
  • DVA回答は「信頼ポリシー条件」と「最小権限」をセットで述べる。

参照: https://docs.aws.amazon.com/STS/latest/APIReference/API_AssumeRoleWithWebIdentity.html

Topic 2: API Gateway Lambda Authorizer と Cognito Authorizer の使い分けAPIアクセス制御

定義

Lambda Authorizerは独自ロジックでリクエスト許可/拒否を判定する認可方式。Cognito AuthorizerはCognitoユーザープールのJWT検証をネイティブに行う方式。

重要性

  • DVAは「認証はできるが認可が弱い」設計ミスを問う問題が多い。
  • レイテンシ、運用コスト、要件変更耐性に直結する選択ポイント。

使い分け

  • 標準JWT検証中心: Cognito Authorizer（実装量が少なく保守しやすい）。
  • ABAC/外部DB参照など独自判定: Lambda Authorizer（柔軟だが遅延増に注意）。
  • 高トラフィック: Authorizerキャッシュを使い、TTLを短すぎ/長すぎにしない。

誤解

  • 「Lambda Authorizerの方が常に高機能で優位」→ 要件が単純なら過剰設計。
  • 「認証OKなら全API呼び出しOK」→ スコープ・クレーム単位の認可設計が必要。

3分要点

  • 標準要件はCognito、独自要件はLambda Authorizer。
  • 性能要件がある場合はキャッシュ戦略を明示する。
  • DVAでは「認証方式 + 認可粒度 + キャッシュTTL」を答える。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-use-lambda-authorizer.html

Topic 3: Lambda メモリ設定と CPU性能・コスト最適化実行最適化

定義

Lambdaはメモリを増やすと同時に割り当てCPUも増える。実行時間短縮で総コストが下がるケースがあり、単純な「最小メモリ=最安」は成立しない。

重要性

  • DVAでは「コスト最適化」と「パフォーマンス改善」を同時達成する設計が頻出。
  • 特にCPUバウンド処理でメモリ調整の効果が大きい。

使い分け

  • CPUバウンド（暗号化/圧縮/変換）: メモリ増で実行時間短縮を狙う。
  • I/O待ち主体: メモリ増の効果は限定的、接続再利用やバッチサイズ見直し優先。
  • 本番最適化: Lambda Power Tuning等で複数メモリを実測比較。

誤解

  • 「メモリ増は必ず高コスト」→ 実行時間短縮で請求額が下がる場合がある。
  • 「1回のベンチで十分」→ 入力サイズや同時実行で結果が変わる。

3分要点

  • Lambdaはメモリ調整がCPU調整でもある。
  • 最適値は理論ではなく実測で決める。
  • DVA回答は「ワークロード特性 + 実測比較 + 選定理由」を示す。

参照: https://docs.aws.amazon.com/lambda/latest/dg/configuration-memory.html
