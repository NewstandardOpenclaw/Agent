Topic 1: EventBridge 再試行・DLQ と Lambda 非同期失敗処理失敗制御の責務分離

定義

EventBridge ルールのターゲット失敗処理は、ターゲット呼び出し失敗時に再試行し、最終的に DLQ に退避できる機能。Lambda 非同期呼び出し側にも再試行と失敗送信先（OnFailure）があり、層ごとの失敗制御を設計する。

重要性

  • 障害時の「どこで失敗が吸収されるか」を明確化し、イベント欠落を防げる。
  • 再試行の重複設計を避け、過剰リトライによるコストと遅延を抑えられる。

使い分け

  • EventBridge 側で吸収: ターゲットが一時的に不安定で、イベント配送保証を先に確保したい。
  • Lambda 非同期失敗先を重視: 関数内ロジック失敗の分析・再処理フローを詳細に持ちたい。
  • 両方使う場合: どの層の DLQ を一次調査点にするか運用手順を先に固定する。

誤解

  • 「DLQ があれば失敗設計は完了」→ 誤り。再投入手順と責任者がないと詰む。
  • 「再試行を多くすれば信頼性が上がる」→ 誤り。依存先障害を悪化させることがある。

3分要点

  • EventBridge と Lambda 非同期は、失敗制御の“層”が違う。
  • 再試行・DLQ・再投入手順をセットで設計して初めて運用可能。
  • DVAでは「失敗時の行き先」を問う問題が頻出。

参照: https://docs.aws.amazon.com/eventbridge/latest/userguide/eb-rule-dlq.html

Topic 2: Cognito User Pool と Identity Pool の役割分担

定義

User Pool はユーザー認証（サインアップ/サインイン・トークン発行）を担う。Identity Pool は認証済み/未認証ユーザーへ一時 AWS 認証情報（IAM ロール）を払い出す。

重要性

  • 認証と AWS リソースアクセス制御を分離し、最小権限を実装しやすい。
  • モバイル/SPA で S3 や API へ直接アクセスする際の定番設計になる。

使い分け

  • User Pool のみ: アプリ内認証が目的で、AWS リソース直接アクセスが不要。
  • User Pool + Identity Pool: ユーザー属性に応じて IAM ロールを切り替えたい。
  • 外部 IdP 連携: Google/OIDC/SAML の既存ID基盤を利用したい場合に Identity Pool 側で統合。

誤解

  • 「User Pool トークンだけで S3 に直接アクセスできる」→ 誤り。通常は Identity Pool で AWS 資格情報に交換する。
  • 「Identity Pool が認証も全部やる」→ 誤り。本人確認自体は User Pool や外部 IdP の役割。

3分要点

  • User Pool = 認証、Identity Pool = AWS 権限付与。
  • 責務分離を押さえると設問の選択肢を消し込みやすい。
  • DVAでは「どちらを使うべきか」が定番論点。

参照: https://docs.aws.amazon.com/cognito/latest/developerguide/cognito-identity.html

Topic 3: CodeBuild の buildspec と環境変数の安全運用CIの秘密情報管理

定義

buildspec.yml は CodeBuild のフェーズ（install/pre_build/build/post_build）と成果物を定義するファイル。機密値は平文埋め込みせず、Parameter Store/Secrets Manager から注入する。

重要性

  • ビルド失敗時の切り分け速度が上がり、デプロイ品質が安定する。
  • CIログ経由のシークレット漏えいリスクを減らせる。

使い分け

  • Parameter Store: 階層管理・比較的軽量な設定値を扱う。
  • Secrets Manager: ローテーション前提の認証情報を扱う。
  • 環境変数の直書き: 機密でない定数のみ。キーやトークンは避ける。

誤解

  • 「マスク表示されるからログに出しても安全」→ 誤り。周辺ログや失敗時出力で漏れる可能性がある。
  • 「buildspec は1回作れば不変」→ 誤り。依存更新やセキュリティ要件変更で継続保守が必要。

3分要点

  • buildspec は“ビルド手順書”であり、運用品質を左右する。
  • 機密値は必ずマネージドストア経由で注入する。
  • DVAでは CI/CD と IAM/Secrets の組み合わせ理解が得点源。

参照: https://docs.aws.amazon.com/codebuild/latest/userguide/build-spec-ref.html
