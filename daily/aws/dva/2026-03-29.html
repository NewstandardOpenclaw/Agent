Topic 1: DynamoDB 条件付き書き込み（ConditionExpression）二重更新防止

定義

ConditionExpression は、指定した条件が真のときだけ PutItem / UpdateItem / DeleteItem を実行する機能。楽観ロック（version属性）や重複登録防止（attribute_not_exists）を実装できる。

重要性

  • 同時更新時のデータ破壊を防ぐため、DVAで非常に頻出。
  • 「Lambda再試行で同じ更新が二重実行された」ケースの防止策として実務でも必須。

使い分け

  • attribute_not_exists(PK): 初回作成のみ許可（重複作成防止）。
  • version = :expected: 楽観ロックで競合更新を検知。
  • TransactWriteItems: 複数アイテムの整合性を同時に守りたい場合。

誤解

  • 「UpdateItemなら上書きしても問題ない」→ 誤り。競合で意図しない値になる。
  • 「条件失敗はシステム障害」→ 誤り。業務上あり得る競合としてハンドリングすべき。

3分要点

  • DynamoDBの安全更新は ConditionExpression が基本。
  • 失敗時は ConditionalCheckFailedException を前提に再取得・再試行を設計。
  • 単一アイテムなら条件付き更新、複数整合性ならトランザクションを検討。

参照: https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/Expressions.ConditionExpressions.html

Topic 2: Lambda イベントソースマッピングの部分バッチ失敗（SQS）

定義

SQS をトリガーにした Lambda では、Partial Batch Response を使うと「失敗したメッセージIDのみ再試行」できる。成功分まで巻き戻して再処理する無駄を減らせる。

重要性

  • 高スループット処理でコスト削減・遅延抑制に直結。
  • DVA試験の「再試行最適化」「冪等性」「DLQ設計」の複合問題に強い。

使い分け

  • 全件失敗: 実装は単純だが再試行コストが高い。
  • 部分バッチ失敗: 失敗分のみ再処理、実務向き。
  • DLQ/最大受信回数: 永続失敗の隔離と運用監視に必須。

誤解

  • 「1件失敗したらバッチ全体を失敗にすべき」→ 常に正解ではない。
  • 「部分バッチ失敗なら冪等性不要」→ 誤り。重複処理は依然起こり得る。

3分要点

  • SQS+Lambda は at-least-once 前提で冪等性必須。
  • Partial Batch Response で再試行対象を最小化できる。
  • DLQとCloudWatchメトリクスで失敗の見える化をセット運用。

参照: https://docs.aws.amazon.com/lambda/latest/dg/with-sqs.html

Topic 3: Cognito JWT 検証と API Gateway Authorizer の責務分離

定義

Cognito User Pool で発行した JWT（ID/Access Token）を API Gateway Authorizer で検証し、アプリ本体では業務ロジックに集中する設計。認証・認可を入口で統制できる。

重要性

  • セキュリティ境界をAPI層に集約し、実装漏れを減らせる。
  • DVAで頻出の「最小実装で安全なAPI公開」系設問に直結。

使い分け

  • Cognito Authorizer: User Pool利用時の標準解。
  • Lambda Authorizer: 独自トークン形式や複雑な判定が必要な場合。
  • IAM認可: AWS署名付きリクエスト中心の内部APIに適合。

誤解

  • 「IDトークンをAPI呼び出しに使う」→ 原則は Access Token を使う。
  • 「Authorizerがあれば細かい権限判定は不要」→ 誤り。スコープ/クレームの業務判定は必要。

3分要点

  • 認証は入口（API Gateway）で実施し、責務を分離。
  • JWTの種類と用途（ID/Access）を混同しない。
  • Authorizer + 最小権限設計で運用しやすいAPIにする。

参照: https://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-integrate-with-cognito.html
