React Day 1: コンポーネントの基本（JSXでUIを部品化）

解説:
Reactは「画面を部品（コンポーネント）」として組み立てる考え方。
まずは function Greeting(){ return <h1>Hello</h1> } のように、1つの表示部品を作るところから始める。
見た目が同じUIを再利用できるので、変更コストが一気に下がる。

なぜ面白い？
HTMLを「静的なページ」ではなく、「再利用できるUIの部品集」として扱えるようになるから。
画面が大きくなるほど、Reactの真価（保守性・拡張性）が見えてくる。

You向けミニ課題（10分）:

  • Greeting コンポーネントを作って、名前を表示してみる
  • App から Greeting を2回呼び出して「再利用」を体験する
  • 最後に「どこが再利用しやすいか」を1行メモ
